## Report Management

### Report Types

Users can report content for:

| Report Type | Description | Typical Action |
|-------------|-------------|----------------|
| **Harassment** | Bullying, threats, personal attacks | Hide post, warn user |
| **Hate Speech** | Discrimination, slurs, bigotry | Hide immediately |
| **Spam** | Repetitive, promotional content | Remove post |
| **Self-Harm** | Suicide ideation, self-injury | Crisis intervention |
| **Personal Info** | Names, contacts, locations | Hide immediately |
| **Sexual Content** | Inappropriate sexual material | Hide, possible ban |
| **Violence** | Threats, graphic content | Hide immediately |
| **Misinformation** | False academic/health info | Blur with warning |
| **Other** | Custom reason provided | Case-by-case |

### Reviewing Reports

**Step-by-Step Process:**

1. **Read the Report**
   - Review the reporter's reason and description
   - Note the report type and timestamp
   - Check if multiple users reported the same content

2. **Examine the Content**
   - View the reported post or comment
   - Read full context (thread, replies)
   - Consider author's intent and community impact

3. **Check Guidelines**
   - Does it violate community standards?
   - Is it potentially harmful?
   - Could it be educational if context is added?

4. **Make a Decision**
   - **Mark Valid** - Violation confirmed, action taken
   - **Mark Invalid** - No violation, content restored

5. **Take Action** (if valid)
   - Blur for mild violations
   - Hide for serious violations
   - Restore if mistakenly flagged

### Automatic Actions

SafeVoice has built-in thresholds:

| Reports | Automatic Action | Your Role |
|---------|------------------|-----------|
| **3 reports** | Content blurred | Review within 24h |
| **5 reports** | Post hidden | Confirm or restore |
| **10 reports** | Post deleted | Investigation required |

**Important:** Always review automatically-actioned content. False reporting does happen.

### Valid Report Workflow

When you mark a report as **Valid**:

1. **Post is hidden** (if not already)
2. **Reporter earns +10 VOICE** (encourages good reporting)
3. **You earn +30 VOICE** (moderator action reward)
4. **Notification sent** to reporter confirming validation
5. **Post status updated** to "Hidden by moderator"

### Invalid Report Workflow

When you mark a report as **Invalid**:

1. **Content is restored** (blur/hide removed)
2. **Reporter loses -5 VOICE** (discourages false reports)
3. **You earn +30 VOICE** (moderator action reward)
4. **Notification sent** to reporter explaining decision
5. **Post returns to normal visibility**

## Content Moderation

### Moderator Actions

You can take these actions on any post or comment:

#### 1. Blur Post
**When to use:**
- Sensitive but not harmful content
- Potential triggers (mental health, trauma)
- Content needing context or warnings
- Borderline guideline violations

**How to blur:**
1. Click three dots (‚ãÆ) on post
2. Select **"Blur Post"**
3. Add reason (shown to users)
4. Confirm action

**Effect:**
- Post content hidden behind blur screen
- Warning message displayed
- User must click "View Content" to see
- Original poster notified

#### 2. Hide Post
**When to use:**
- Clear guideline violations
- Harmful or dangerous content
- Repeated violations by user
- Content requiring investigation

**How to hide:**
1. Click three dots (‚ãÆ) on post
2. Select **"Hide Post"**
3. Add reason for records
4. Confirm action

**Effect:**
- Post removed from public feed
- Still accessible to moderators for review
- Original poster notified
- Can be restored if mistake

#### 3. Verify Advice
**When to use:**
- Exceptionally helpful comments
- Accurate, well-researched information
- Compassionate crisis support
- Expert-level advice

**How to verify:**
1. Find the outstanding comment
2. Click **"Mark as Verified Advice"**
3. Confirm verification

**Effect:**
- Comment gets ‚úì Verified badge
- Author earns **+200 VOICE**
- Comment highlighted in thread
- Builds trust in advice

**Criteria for verification:**
- Factually accurate
- Genuinely helpful
- Well-explained and clear
- Appropriate for situation

#### 4. Restore Post
**When to use:**
- False reports confirmed
- Content mistakenly hidden
- Situation resolved
- Appeals granted

**How to restore:**
1. Find the hidden post
2. Click **"Restore Post"**
3. Confirm restoration

**Effect:**
- Post returns to feed
- Blur/hide removed
- Normal visibility restored
- Original poster notified

### Pin Posts (Future Feature)

**When to pin:**
- Important community announcements
- Safety resources and helplines
- Event information
- Guidelines reminders

## Crisis Intervention

### Identifying Crisis Posts

SafeVoice automatically flags posts containing:
- Suicide ideation keywords
- Self-harm expressions
- Abuse disclosures
- Severe mental health crises

**Visual indicators:**
- üö® Red "Crisis" badge on post
- Warning banner above content
- Helpline resources displayed
- Priority in moderator queue

### Crisis Response Protocol

**Immediate Actions (< 5 minutes):**

1. **Verify the Crisis**
   - Read the full post carefully
   - Assess severity (high/critical)
   - Check for immediate danger signs

2. **Ensure Resources are Visible**
   - Crisis modal should auto-display
   - Helplines should be prominent
   - Support options clearly shown

3. **Monitor for Responses**
   - Watch for supportive comments
   - Flag inappropriate responses
   - Verify advice quality

**Follow-Up Actions (< 1 hour):**

4. **Engage Support**
   - Add a supportive moderator comment
   - Encourage professional help-seeking
   - Provide additional resources if needed

5. **Track Resolution**
   - Monitor original poster's responses
   - Award crisis support rewards
   - Document case for records

### Crisis Keywords

Posts are auto-flagged for:
- "suicide", "kill myself", "end it all"
- "self-harm", "cutting", "hurting myself"
- "abuse", "assault", "unsafe"
- "no reason to live", "better off dead"

### What NOT to Do

‚ùå **Don't** attempt to identify the user
‚ùå **Don't** offer direct counseling
‚ùå **Don't** dismiss or minimize feelings
‚ùå **Don't** promise confidentiality you can't keep
‚ùå **Don't** delay escalation if needed

### When to Escalate

**Contact Platform Administrators if:**
- Imminent threat to life
- Child abuse disclosure
- Ongoing assault or danger
- User provides specific suicide plan
- Campus emergency services needed

| Verify advice | 30 VOICE | 5 min | Moderator action |
| Restore post | 30 VOICE | 5 min | Moderator action |

### Cooldown System

To prevent reward farming:
- **5-minute cooldown** between moderator actions
- Cooldown applies per moderator, not per action type
- Action still completes even if reward is skipped
- Encourages thoughtful, spaced moderation

### Reward Tracking

Your moderation rewards appear in:
- Transaction history (Wallet section)
- Earnings breakdown (bonuses category)
- Moderator panel (action count)

## Best Practices

### Decision-Making Framework

**For every moderation decision, ask:**

1. **Is it harmful?** - Could it hurt someone?
2. **Is it a violation?** - Clear guideline breach?
3. **Is context important?** - Does situation matter?
4. **Is action proportionate?** - Fitting response?
5. **Is it reversible?** - Can I undo if wrong?

### Communication Guidelines

**When commenting as moderator:**
- ‚úÖ Be clear and specific
- ‚úÖ Explain your reasoning
- ‚úÖ Provide resources when relevant
- ‚úÖ Maintain professional tone
- ‚ùå Don't be condescending
- ‚ùå Don't engage in arguments
- ‚ùå Don't make it personal

### Bias Awareness

**Watch for unconscious bias:**
- Treatment based on posting history
- Assumptions about user intent
- Different standards for different topics
- Personal triggers influencing decisions

**Mitigation strategies:**
- Review guidelines before each decision
- Take breaks during difficult content
- Consult other moderators when uncertain
- Document reasoning for transparency

### Self-Care

Moderating can be emotionally draining:

- üßò **Take regular breaks** - Step away between reviews
- üó£Ô∏è **Debrief with peers** - Talk to other moderators
- üö´ **Know your limits** - Skip content that triggers you
- üõå **Rest adequately** - Don't moderate while tired
- üíô **Seek support** - Use crisis resources yourself if needed

## Escalation Procedures

### When to Escalate

Escalate to administrators if:

1. **Imminent Danger**
   - Specific suicide plan with timeline
   - Active assault or abuse situation
   - User indicates immediate harm

2. **Legal Issues**
   - Child abuse disclosures
   - Credible threats of violence
   - Criminal activity planning
   - Intellectual property violations

3. **Systemic Issues**
   - Coordinated harassment campaign
   - Platform exploitation or gaming
   - Large-scale spam or bot activity
   - Security vulnerabilities discovered

4. **Moderator Conflicts**
   - Decision uncertainty on serious matters
   - User disputes moderator action
   - Conflicting moderator opinions
   - Personal conflict of interest

### Escalation Process

**Step-by-step:**

1. **Document the Issue**
   - Post/comment ID
   - User ID(s) involved
   - Timeline of events
   - Actions already taken
   - Specific concerns

2. **Use Escalation Channels**
   - [To be configured per deployment]
   - Include all documentation
   - Mark urgency level

3. **Continue Monitoring**
   - Watch for developments
   - Take interim protective actions
   - Keep records updated

4. **Follow Through**
   - Implement admin decisions
   - Update affected parties
   - Document resolution

## Frequently Asked Questions

### Can I moderate my own posts?
No. Moderator actions should not be used on your own content.

### What if I make a mistake?
Use the Restore action to reverse decisions. Document the correction.

### How do I handle disagreements with users?
Remain professional, explain reasoning once, and don't engage in arguments.

### Can users appeal moderator decisions?
Yes. Users can request review through the reporting system.

### What if another moderator disagrees with me?
Discuss respectfully. If unresolved, escalate to administrators.
